    Metoda Object.Equals (Object)

    .NET Framework (current version)
    Ostatní verze

    • .NET Framework 4
     

    Určuje, zda zadaný objekt je rovna aktuálního objektu.

    Návratová hodnota

    Poznámky

    Typ porovnání mezi aktuální instance a obj parametr závisí na tom, zda aktuální instance typu odkazu nebo typu hodnoty.

    • Pokud je aktuální instance typu odkazu Equals(Object) Metoda testy pro rovnost reference a volání Equals(Object) Metoda je ekvivalentní volání ReferenceEquals Metoda. Referenční rovnost znamená, že proměnné objektu, které jsou porovnány odkazovat na stejný objekt. Následující příklad ukazuje výsledku takové porovnání. Definuje Person Třída, která je typem odkazu, a volání Person konstruktor třídy pro vytvoření instance dva nové Person objekty, person1a a person2, které mají stejnou hodnotu. Také přiřadí person1a jiné proměnné objektu person1b. Jako výstup z příkladu person1a a person1b jsou stejné, protože odkazují na stejný objekt. Však person1a a person2 nejsou shodné, i když mají stejnou hodnotu.

      C#
      VB
      Zkopírovat 
      using System;

// Define a reference type that does not override Equals.
public class Person
{
   private string personName;

   public Person(string name)
   {
      this.personName = name;
   }

   public override string ToString()
   {
      return this.personName;
   }
}

public class Example
{
   public static void Main()
   {
      Person person1a = new Person("John");
      Person person1b = person1a;
      Person person2 = new Person(person1a.ToString());

      Console.WriteLine("Calling Equals:"); 
      Console.WriteLine("person1a and person1b: {0}", person1a.Equals(person1b));               
      Console.WriteLine("person1a and person2: {0}", person1a.Equals(person2));  

      Console.WriteLine("\nCasting to an Object and calling Equals:");
      Console.WriteLine("person1a and person1b: {0}", ((object) person1a).Equals((object) person1b));
      Console.WriteLine("person1a and person2: {0}", ((object) person1a).Equals((object) person2)); 
   }
}

//       person1a and person1b: True
//       person1a and person2: False
//       
//       Casting to an Object and calling Equals:
//       person1a and person1b: True
//       person1a and person2: False

    Protože Object Třída je základní třída pro všechny typy v rozhraní .NET Framework Object.Equals(Object) Metoda poskytuje porovnání rovnosti výchozí pro všechny ostatní typy. Však často přepsat typy Equals Metoda k implementaci rovnosti hodnoty. Další informace najdete v poznámkách k pro volající a poznámky pro dědice oddíly.

    Poznámky pro prostředí Windows Runtime

    Při volání Equals(Object) přetížení metody třídy v prostředí Windows Runtime, poskytuje výchozí chování pro třídy, které nechcete přepsat Equals(Object). Jedná se o součást podpory, která poskytuje rozhraní .NET Framework prostředí Windows Runtime (viz Podpora pro aplikace pro web Windows Store a prostředí Windows Runtime v rozhraní .NET Framework). Třídy v prostředí Windows Runtime nedědí Object, a není implementovat Equals(Object)Metoda. Zobrazují se však mít ToString, Equals(Object), a GetHashCode metody při použití v kódu jazyka C# nebo Visual Basic a .NET Framework poskytuje výchozí chování pro tyto metody.

    Poznámka

    prostředí Windows Runtime třídy, které jsou napsány v jazyce C# nebo Visual Basic můžete přepsat Equals(Object) přetížení metody.

    Poznámky pro volající

    Odvozené třídy často přepsat Object.Equals(Object) Metoda k implementaci rovnosti hodnoty. Kromě toho typy také často poskytují další silného typu přetížení pro Equals Metoda obvykle implementací IEquatable<T> rozhraní. Při volání Equals Metoda testování rovnosti, byste měli vědět, zda aktuální instance přepsání Object.Equals a pochopit, jak konkrétní volání Equals Metoda je vyřešený. V opačném případě může provedení testu rovnosti, které se liší od vašich představ a metoda může vrátit neočekávanou hodnotu.

    Následující příklad uvádí ukázku. Ji vytvoří tři StringBuilder objekty s shodné řetězce a poté volá čtyři Equals metody. První metoda volání vrátí true, a zbývající tři návrat false.

    C#
    VB
    Zkopírovat 
    using System;
using System.Text;

public class Example
{
   public static void Main()
   {
      StringBuilder sb1 = new StringBuilder("building a string...");
      StringBuilder sb2 = new StringBuilder("building a string...");

      Console.WriteLine("sb1.Equals(sb2): {0}", sb1.Equals(sb2));
      Console.WriteLine("((Object) sb1).Equals(sb2): {0}", 
                        ((Object) sb1).Equals(sb2));
      Console.WriteLine("Object.Equals(sb1, sb2): {0}",
                        Object.Equals(sb1, sb2));      

      Object sb3 = new StringBuilder("building a string...");
      Console.WriteLine("\nsb3.Equals(sb2): {0}", sb3.Equals(sb2));                              
   }
}
// The example displays the following output:
//       sb1.Equals(sb2): True
//       ((Object) sb1).Equals(sb2): False
//       Object.Equals(sb1, sb2): False
//
//       sb3.Equals(sb2): False

    V případě, první silného typu StringBuilder.Equals(StringBuilder) přetížení metody, které testy pro rovnost hodnot, je volána. Protože řetězce přiřazen dvou StringBuilder jsou objekty rovnocenné, metoda vrátí true. Však StringBuilder nepřepisuje Object.Equals(Object). Z tohoto důvodu při StringBuilder objekt je přetypovat na Object, když StringBuilder instance je přiřazen proměnné typu Object, a kdy Object.Equals(Object, Object) metodě je předána dva StringBuilder objekty, výchozí Object.Equals(Object) Metoda je volána. Protože StringBuilder je typem odkazu, jedná se o ekvivalent předání dvou StringBuilder objekty k ReferenceEquals Metoda. Ačkoliv všechny tři StringBuilder objekty obsahují shodné řetězce, které odkazují na tři odlišné objekty. V důsledku toho tyto tři metoda volá návratový false.

    Můžete porovnat aktuální objekt k jinému objektu pro referenční rovnost voláním ReferenceEquals Metoda. V jazyce Visual Basic, můžete použít také is klíčové slovo (například If Me Is otherObject Then ...).

    Typ hodnoty, měli byste vždy přepsat Equals, protože testy pro rovnost, které závisí na reflexi nabízejí nízký výkon. Můžete také přepsat výchozí implementace Equals pro odkazové typy pro testování rovnosti hodnoty namísto referenční rovnost a definování Přesný význam rovnosti hodnoty. Tato implementace Equals vrátit true Pokud dva objekty mají stejnou hodnotu, i v případě, že se nepoužívají stejnou instanci. Typ implementátor rozhodne, co představuje hodnotu objektu, ale je obvykle některá nebo všechna data uložená v proměnné instance objektu. Například hodnota String objektu je založena na znaky řetězce; String.Equals(Object) přepsání metody Object.Equals(Object) Metoda vrátí true pro jakékoli dva řetězce instancí, které obsahují stejné znaky ve stejném pořadí.

    Následující příklad ukazuje, jak přepsat Object.Equals(Object) Metoda pro testování rovnosti hodnoty. Přepíše Equals metodu pro Person třídy. Pokud Person přijmout jeho implementace základní třídy rovnosti, dva Person objektů by byla stejná pouze v případě, že se odkazuje jeden objekt. V tomto případě však dva Person jsou objekty rovnocenné, pokud mají stejnou hodnotu pro Person.Id vlastnost.

    C#
    VB
    Zkopírovat 
    public class Person
{
   private string idNumber;
   private string personName;

   public Person(string name, string id)
   {
      this.personName = name;
      this.idNumber = id;
   }

   public override bool Equals(Object obj)
   {
      Person personObj = obj as Person; 
      if (personObj == null)
         return false;
      else
         return idNumber.Equals(personObj.idNumber);
   }

   public override int GetHashCode()
   {
      return this.idNumber.GetHashCode(); 
   }
}

public class Example
{
   public static void Main()
   {
      Person p1 = new Person("John", "63412895");
      Person p2 = new Person("Jack", "63412895");
      Console.WriteLine(p1.Equals(p2));
      Console.WriteLine(Object.Equals(p1, p2));
   }
}
// The example displays the following output:
//       True
//       True

    Kromě potlačení Equals, můžete implementovat IEquatable<T> pro poskytnutí silného typu testování rovnosti.

    Následující příkazy musí být splněny pro všechny implementace Equals(Object) Metoda. V seznamu x, y, a z představují odkazy na objekty, které nejsou null.

    • x.Equals(x) Vrátí true, s výjimkou případů, které zahrnují typy s plovoucí desetinnou čárkou. Viz ISO/IEC/IEEE 60559:2011, informačních technologií--mikroprocesoru systémy--aritmetické operace s plovoucí desetinnou čárkou.

    • x.Equals(y) vrátí stejnou hodnotu jako y.Equals(x).

    • x.Equals(y) Vrátí true Pokud obě x a y jsou NaN.

    • Pokud (x.Equals(y) && y.Equals(z)) vrátí true, pak x.Equals(z) vrátí true.

    • Následná volání x.Equals(y) vrátit stejnou hodnotu, dokud odkazují objekty x a y se nezmění.

    • x.Equals(null) Vrátí false.

    Implementace Equals nesmí vyvolat výjimky; by vždy vrátit hodnotu. Například pokud obj je null, Equals Metoda by měla vrátit false namísto vyvolání ArgumentNullException.

    Postupujte podle následujících pokynů při přepisování Equals(Object):

    • Typy, které implementují IComparable nutné přepsat Equals(Object).

    • Typy toto přepsání Equals(Object) musí také přepsat GetHashCodejinak zatřiďovacích tabulek nemusí fungovat správně.

    • Měli byste zvážit implementaci IEquatable<T> rozhraní pro podporu silně typované testy pro rovnost. Vaše IEquatable<T>.Equals implementace by měla vrátit výsledky, které jsou v souladu s Equals.

    • Pokud váš programovací jazyk podporuje přetížení operátoru a přetížení operátoru rovnosti pro daný typ, je nutné přepsat Equals(Object) Metoda vrátí stejný výsledek jako operátor rovnosti. To pomáhá zajistit, že třída knihovny kódu, který používá Equals (jako je například ArrayList a Hashtable) se chová způsobem, který je konzistentní s způsob, jak se používá operátor rovnosti kódem aplikace.

    Pokyny pro typy odkazů

    Následující pokyny se vztahují k přepsání Equals(Object) pro typ odkazu:

    • Vezměte v úvahu přepsání Equals Pokud sémantiku typu jsou na základě skutečnosti, že typ představuje některé hodnoty.

    • Většina odkazové typy nesmí přetížení operátoru rovnosti, i v případě, že má přednost před Equals. Při implementaci typu odkaz, který je určen k mají sémantiku hodnotu, jako je například typ komplexního čísla, je však nutné přepsat operátor rovnosti.

    • Neměli přepsat Equals na proměnlivé odkazového typu. Důvodem je, že přepsání Equals vyžaduje, aby také přepsat GetHashCode metodu, jak je popsáno v předchozí části. To znamená, že kód hash instance typu proměnlivé odkazové lze změnit během celé jeho životnosti, což může způsobit objekt, který má dojít ke ztrátě v tabulce hash.

    Pokyny pro typy hodnot

    Následující pokyny se vztahují k přepsání Equals(Object) pro typ hodnoty:

    • Pokud definujete hodnotový typ, který obsahuje jeden nebo více polí, jejichž hodnoty jsou typy odkazů, byste měli přepsat Equals(Object). Equals(Object) Implementace poskytovaná ValueType provádí porovnání bajt po bajtu pro typy hodnota, jejíž pole jsou všechny typy hodnot, ale používá reflexi pro provedení porovnání pole pole typů hodnot, jejichž pole obsahují typy odkazů.

    • Pokud přepíšete Equals a váš jazyk vývoj podporuje přetížení operátoru, musí přetížení operátoru rovnosti.

    • Měli byste implementovat IEquatable<T> rozhraní. Volání silného typu IEquatable<T>.Equals metoda zabraňuje zabalení obj argument.

    Příklady

    Následující příklad ukazuje Point třídu, která přepíše Equals Metoda poskytnout rovnosti hodnoty a Point3D Třída, která je odvozena od Point. Protože Point přepsání Object.Equals(Object) pro testování rovnosti hodnoty Object.Equals(Object) není zavolána metoda. Však Point3D.Equals volání Point.Equals protože Point implementuje Object.Equals(Object) způsobem, který poskytuje rovnosti hodnoty.

    C#
    VB
    Zkopírovat 
    using System;

class Point
{
   protected int x, y;

   public Point() : this(0, 0) 
   { }

   public Point(int x, int y) 
   {
      this.x = x;
      this.y = y;
   }

   public override bool Equals(Object obj) 
   {
      //Check for null and compare run-time types.
      if ((obj == null) || ! this.GetType().Equals(obj.GetType())) {
         return false;
      }
      else { 
         Point p = (Point) obj; 
         return (x == p.x) && (y == p.y);
      }   
   }

   public override int GetHashCode() 
   {
      return (x << 2) ^ y;
   }

    public override string ToString()
    {
        return String.Format("Point({0}, {1})", x, y);
    }
}

sealed class Point3D: Point 
{
   int z;

   public Point3D(int x, int y, int z) : base(x, y) 
   {
      this.z = z; 
   }

   public override bool Equals(Object obj) 
   {
      Point3D pt3 = obj as Point3D;
      if (pt3 == null)
         return false;
      else
         return base.Equals((Point)obj) && z == pt3.z;
   }

   public override int GetHashCode() 
   {
      return (base.GetHashCode() << 2) ^ z;
   }

   public override String ToString() 
   {
        return String.Format("Point({0}, {1}, {2})", x, y, z);
    }
}

class Example 
{
  public static void Main() 
  {
     Point point2D = new Point(5, 5);
     Point3D point3Da = new Point3D(5, 5, 2);
     Point3D point3Db = new Point3D(5, 5, 2);
     Point3D point3Dc = new Point3D(5, 5, -1);

     Console.WriteLine("{0} = {1}: {2}", 
                       point2D, point3Da, point2D.Equals(point3Da));
     Console.WriteLine("{0} = {1}: {2}", 
                       point2D, point3Db, point2D.Equals(point3Db));        
     Console.WriteLine("{0} = {1}: {2}", 
                       point3Da, point3Db, point3Da.Equals(point3Db));
     Console.WriteLine("{0} = {1}: {2}", 
                       point3Da, point3Dc, point3Da.Equals(point3Dc));
  } 
}
// The example displays the following output:
//       Point(5, 5) = Point(5, 5, 2): False
//       Point(5, 5) = Point(5, 5, 2): False
//       Point(5, 5, 2) = Point(5, 5, 2): True
//       Point(5, 5, 2) = Point(5, 5, -1): False

    The Point.Equals method checks to make sure that the obj argument is not null and that it references an instance of the same type as this object. If either check fails, the method returns false.

    The Point.Equals method calls the M:System.Object.GetType method to determine whether the run-time types of the two objects are identical. If the method used a check of the form obj is Point in C# or TryCast(obj, Point) in Visual Basic, the check would return true in cases where obj is an instance of a derived class of Point, even though obj and the current instance are not of the same run-time type. Having verified that both objects are of the same type, the method casts obj to type Point and returns the result of comparing the instance fields of the two objects.

    In Point3D.Equals, the inherited Point.Equals method, which overrides M:System.Object.Equals(System.Object), is invoked before anything else is done. Because Point3D is a sealed class (NotInheritable in Visual Basic), a check in the form obj is Point in C# or TryCast(obj, Point) in Visual Basic is adequate to ensure that obj is a Point3D object. If it is a Point3D object, it is cast to a Point object and passed to the base class implementation of M:System.Object.Equals(System.Object). Only when the inherited Point.Equals method returns true does the method compare the z instance fields introduced in the derived class.

    Následující příklad definuje Rectangle třídu, která implementuje interně obdélník jako dva Point objekty. Rectangle Třídy také přepsání Object.Equals(Object) stanovit rovnosti hodnoty.

    C#
    VB
    Zkopírovat 
    using System;

class Rectangle 
{
   private Point a, b;

   public Rectangle(int upLeftX, int upLeftY, int downRightX, int downRightY) {
      this.a = new Point(upLeftX, upLeftY);
      this.b = new Point(downRightX, downRightY);
   }

   public override bool Equals(Object obj) {
      // Perform an equality check on two rectangles (Point object pairs).
      if (obj == null || GetType() != obj.GetType()) 
          return false;
      Rectangle r = (Rectangle)obj;
      return a.Equals(r.a) && b.Equals(r.b);
   }

   public override int GetHashCode() {
      return Tuple.Create(a, b).GetHashCode();
   }

    public override String ToString() 
    {
       return String.Format("Rectangle({0}, {1}, {2}, {3})",
                            a.x, a.y, b.x, b.y); 
    }
}

class Point 
{
  internal int x;
  internal int y;

  public Point(int X, int Y) {
     this.x = X;
     this.y = Y;
  }

  public override bool Equals (Object obj) {
     // Performs an equality check on two points (integer pairs).
     if (obj == null || GetType() != obj.GetType()) return false;
     Point p = (Point)obj;
     return (x == p.x) && (y == p.y);
  }

  public override int GetHashCode() {
     return Tuple.Create(x, y).GetHashCode();
  }
}

Obvykle to lze provést napsáním Equals(Object) Metoda z hlediska rovnosti přetíženého operátoru, jako v následujícím příkladu.

    using System;

public struct Complex {
   public double re, im;

   public override bool Equals(Object obj) {
      return obj is Complex && this == (Complex)obj;
   }

   public override int GetHashCode() {
      return Tuple.Create(re, im).GetHashCode();
   }

   public static bool operator ==(Complex x, Complex y) {
      return x.re == y.re && x.im == y.im;
   }

   public static bool operator !=(Complex x, Complex y) {
      return !(x == y);
   }

    public override String ToString()
    {
       return String.Format("({0}, {1})", re, im);
    } 
}

class MyClass 
{
  public static void Main() 
  {
    Complex cmplx1, cmplx2;

    cmplx1.re = 4.0;
    cmplx1.im = 1.0;

    cmplx2.re = 2.0;
    cmplx2.im = 1.0;

    Console.WriteLine("{0} <> {1}: {2}", cmplx1, cmplx2, cmplx1 != cmplx2);        
    Console.WriteLine("{0} = {1}: {2}", cmplx1, cmplx2, cmplx1.Equals(cmplx2));        

    cmplx2.re = 4.0;

    Console.WriteLine("{0} = {1}: {2}", cmplx1, cmplx2, cmplx1 == cmplx2);        
    Console.WriteLine("{0} = {1}: {2}", cmplx1, cmplx2, cmplx1.Equals(cmplx2));          
  }
}

//       (4, 1) <> (2, 1): True
//       (4, 1) = (2, 1): False
//       (4, 1) = (4, 1): True
//       (4, 1) = (4, 1): True

    Because Complex is a value type, it cannot be derived from. Therefore, the override to M:System.Object.Equals(System.Object) method need not call M:System.Object.GetType to determine the precise run-time type of each object, but can instead use the is operator in C# or the TypeOf operator in Visual Basic to check the type of the obj parameter.
